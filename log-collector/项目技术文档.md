# 分布式多线程网络日志收集系统 - 项目技术文档

## 1. 项目概述

### 1.1 项目背景

在现代分布式系统中，日志收集和管理是系统监控和故障排查的重要组成部分。传统的日志收集方式存在以下问题：
- 日志分散在各个服务器上，难以集中管理
- 日志量大，处理效率低
- 网络不稳定时，日志可能丢失
- 缺乏统一的日志过滤和检索机制

### 1.2 项目目标

本项目旨在设计和实现一个分布式多线程网络日志收集系统，具有以下特点：
- 支持多客户端、多协议（TCP/UDP）日志采集
- 高并发处理能力，使用多线程和队列机制
- 断点续传，确保日志不丢失
- 灵活的日志过滤和分类机制
- 高效的日志存储和检索

### 1.3 技术栈

- **开发语言**：C语言
- **网络协议**：TCP/UDP
- **数据库**：MySQL 8.0
- **并发处理**：POSIX线程
- **队列机制**：自定义链表队列
- **配置管理**：文件配置

## 3. 核心模块设计

### 3.1 日志采集客户端

#### 3.1.1 功能设计

- **API接口**：提供 `void log_send(enum log_level level, const char* module, const char* content)` 接口
- **日志格式**：「[时间戳][IP][模块名][级别]内容」
- **协议支持**：TCP和UDP
- **缓存机制**：缓存最近100条日志，服务端上线后自动重发
- **配置管理**：支持配置文件设置，无需重新编译

#### 3.1.2 实现细节

- 使用 `socket()` 创建套接字，根据配置选择TCP或UDP
- TCP模式下使用 `connect()` 建立连接，UDP模式下直接发送
- 发送失败时，将日志存入环形缓冲区
- 重连成功后，自动重发缓存中的日志

### 3.2 收集服务端

#### 3.2.1 功能设计

- **多端口监听**：支持TCP（8888）和UDP（8889）端口
- **多线程接收**：主线程创建2个接收线程，分别处理TCP和UDP连接
- **动态过滤**：支持命令行输入过滤规则
- **按模块分类**：将日志按模块名分组

#### 3.2.2 实现细节

- 使用 `socket()`、`bind()`、`listen()` 创建和配置套接字
- TCP线程使用 `accept()` 接受连接，`recv()` 接收数据
- UDP线程使用 `recvfrom()` 接收数据
- 接收到的日志立即存入缓冲队列
- 支持命令行输入「filter WARN ERROR」设置过滤规则

### 3.3 生产消费者并发模块

#### 3.3.1 功能设计

- **缓冲队列**：最大容量1000条，满时阻塞生产者线程
- **消费者线程**：2个线程，从队列取日志并写入MySQL
- **线程安全**：队列操作加互斥锁和条件变量
- **阻塞机制**：队列空时消费者线程阻塞

#### 3.3.2 实现细节

- 使用链表实现队列，支持入队和出队操作
- 使用 `pthread_mutex_t` 保证队列操作的线程安全
- 使用 `pthread_cond_t` 实现生产者和消费者的同步
- 队列满时，生产者线程等待条件变量
- 队列空时，消费者线程等待条件变量

### 3.4 数据存储模块

#### 3.4.1 功能设计

- **分表存储**：按日志级别分表存储（debug_log、info_log、warn_log、error_log）
- **字段设计**：id、timestamp、client_ip、module、content
- **检索方式**：按模块查询、按时间范围查询、模糊查询
- **日志导出**：支持导出到文件

#### 3.4.2 实现细节

- 使用MySQL C API连接数据库
- 根据日志级别选择对应的表
- 构造SQL语句执行插入和查询操作
- 支持导出指定时间范围的日志到文件

### 3.5 配置管理模块

#### 3.5.1 功能设计

- **客户端配置**：protocol、server_ip、server_port
- **服务端配置**：client1_port、client2_port、queue_capacity、consumer_threads
- **配置文件**：client.conf和server.conf
- **默认配置**：配置文件不存在时使用默认值

#### 3.5.2 实现细节

- 逐行读取配置文件，解析键值对
- 支持TCP和UDP协议切换
- 支持端口和队列容量配置
- 配置解析失败时使用默认值

## 4. 数据结构设计

### 4.1 日志级别枚举

```c
typedef enum {
    DEBUG = 0,
    INFO = 1,
    WARN = 2,
    ERROR = 3
} log_level;
```

### 4.2 协议类型枚举

```c
typedef enum {
    PROTOCOL_TCP = 0,
    PROTOCOL_UDP = 1
} protocol_type;
```

### 4.3 日志结构体

```c
typedef struct {
    time_t timestamp;      // 时间戳
    char ip[16];           // 客户端IP
    char module[64];       // 模块名
    log_level level;       // 日志级别
    char content[1024];    // 日志内容
} LogEntry;
```

### 4.4 队列节点结构体

```c
typedef struct QueueNode {
    LogEntry log;          // 日志条目
    struct QueueNode *next; // 下一个节点
} QueueNode;
```

### 4.5 队列结构体

```c
typedef struct {
    QueueNode *head;       // 队首
    QueueNode *tail;       // 队尾
    int size;              // 队列大小
    int capacity;          // 队列容量
    pthread_mutex_t mutex;  // 互斥锁
    pthread_cond_t not_full; // 非满条件变量
    pthread_cond_t not_empty; // 非空条件变量
} LogQueue;
```

### 4.6 客户端配置结构体

```c
typedef struct {
    protocol_type protocol; // 协议类型
    char server_ip[16];     // 服务器IP
    int server_port;        // 服务器端口
    int max_cache;          // 最大缓存数量
} ClientConfig;
```

### 4.7 服务端配置结构体

```c
typedef struct {
    int client1_port;       // 客户端1端口
    int client2_port;       // 客户端2端口
    int queue_capacity;     // 队列容量
    int consumer_threads;   // 消费者线程数
} ServerConfig;
```

## 5. 算法设计

### 5.1 日志发送算法

1. **输入**：日志级别、模块名、日志内容
2. **处理**：
   - 生成时间戳
   - 获取客户端IP
   - 格式化日志
   - 选择协议发送
   - 发送失败时缓存
3. **输出**：发送结果

### 5.2 日志接收算法

1. **输入**：网络数据
2. **处理**：
   - 解析日志格式
   - 检查过滤规则
   - 入队到缓冲队列
3. **输出**：无

### 5.3 生产消费者算法

1. **生产者**：
   - 检查队列是否已满
   - 满则等待，否则入队
   - 通知消费者

2. **消费者**：
   - 检查队列是否为空
   - 空则等待，否则出队
   - 处理日志（写入数据库）
   - 通知生产者

### 5.4 日志过滤算法

1. **输入**：过滤级别
2. **处理**：
   - 解析命令行输入
   - 设置过滤标志
   - 接收日志时检查级别
3. **输出**：过滤结果

### 5.5 日志导出算法

1. **输入**：时间范围、日志级别
2. **处理**：
   - 构造SQL查询
   - 执行查询
   - 写入文件
3. **输出**：导出文件
