## 1. 项目概述

### 1.1 项目背景

在网络文件传输过程中，传统的文件传输工具存在以下问题：

1. **传输中断问题**: 网络不稳定时容易导致传输中断，需要重新传输整个文件
2. **大文件传输困难**: 大文件传输耗时较长，且容易失败
3. **传输效率低**: 单线程传输速度受限
4. **缺乏传输记录**: 无法跟踪传输状态和历史记录

本项目旨在解决这些问题，提供一个支持多线程并发、断点续传的网络文件传输工具。

### 1.2 项目目标

- **断点续传**: 支持传输中断后从断点处继续传输
- **多线程并发**: 提高传输速度和效率
- **大文件支持**: 支持GB级大文件传输
- **传输记录**: 数据库存储传输任务和状态
- **完整性校验**: 使用MD5确保文件传输完整性

### 1.3 技术栈

| 编程语言 | C | C99标准 |

## 5. 算法设计

### 5.1 文件分块算法

```c
int split_file(const char *file_path, long long *file_size, int *total_blocks) {
    struct stat file_stat;
    if (stat(file_path, &file_stat) != 0) {
        return -1;
    }
    
    *file_size = file_stat.st_size;
    *total_blocks = (*file_size + BLOCK_SIZE - 1) / BLOCK_SIZE;
    
    return 0;
}
```

**算法说明**:
- 计算文件大小和总分块数
- 每个块大小固定为4MB
- 最后一个块可能小于4MB

### 5.2 断点续传算法

```c
int resume_transfer(const char *filename, TransferTask *task) {
    // 从数据库获取任务信息
    if (db_get_task(mysql, filename, task) != ERROR_SUCCESS) {
        return -1;
    }
    
    // 检查任务状态
    if (task->status == STATUS_COMPLETED) {
        return 0;  // 文件已完成
    }
    
    return task->completed_blocks;  // 返回已完成的块数
}
```

**算法说明**:
- 从数据库获取传输任务信息
- 检查任务状态和已完成的块数
- 从断点处继续传输

### 5.3 多线程传输算法

```c
void transfer_blocks(int start_block, int end_block, const char *file_path, int sock) {
    ThreadPool *pool = thread_pool_create(5, 10, 20);
    if (pool == NULL) {
        return;
    }
    
    for (int i = start_block; i < end_block; i++) {
        BlockInfo *info = malloc(sizeof(BlockInfo));
        info->block_id = i;
        info->file_path = file_path;
        info->sock = sock;
        
        thread_pool_add_task(pool, transfer_block, info);
    }
    
    thread_pool_destroy(pool);
}
```

**算法说明**:
- 创建线程池
- 为每个文件块创建传输任务
- 提交任务到线程池并发处理
- 等待所有任务完成

### 5.4 MD5校验算法

```c
char *calculate_md5(const char *file_path) {
    FILE *file = fopen(file_path, "rb");
    if (file == NULL) {
        return NULL;
    }
    
    MD5_CTX md5_ctx;
    MD5_Init(&md5_ctx);
    
    char buffer[4096];
    int bytes_read;
    while ((bytes_read = fread(buffer, 1, sizeof(buffer), file)) > 0) {
        MD5_Update(&md5_ctx, buffer, bytes_read);
    }
    
    unsigned char md5[16];
    MD5_Final(md5, &md5_ctx);
    
    static char md5_str[33];
    for (int i = 0; i < 16; i++) {
        sprintf(&md5_str[i*2], "%02x", md5[i]);
    }
    
    fclose(file);
    return md5_str;
}
```

**算法说明**:
- 使用OpenSSL库计算文件MD5值
- 分块读取文件，避免内存溢出
- 返回32位十六进制MD5字符串

### 5.5 网络传输算法

```c
int send_file_block(int sock, int block_id, const char *file_path, long long offset, int block_size) {
    FILE *file = fopen(file_path, "rb");
    if (file == NULL) {
        return -1;
    }
    
    if (fseek(file, offset, SEEK_SET) != 0) {
        fclose(file);
        return -1;
    }
    
    char buffer[BLOCK_SIZE];
    int bytes_read = fread(buffer, 1, block_size, file);
    fclose(file);
    
    if (bytes_read != block_size) {
        return -1;
    }
    
    Command cmd;
    strcpy(cmd.type, "BLOCK");
    strcpy(cmd.filename, basename((char *)file_path));
    cmd.block_id = block_id;
    cmd.block_size = block_size;
    memcpy(cmd.data, buffer, block_size);
    
    if (send(sock, &cmd, sizeof(Command), 0) != sizeof(Command)) {
        return -1;
    }
    
    return 0;
}
```

**算法说明**:
- 定位文件偏移量
- 读取文件块数据
- 构造传输命令
- 发送数据到客户端

### 6.3 命令解析实现

```c
int parse_command(const char *buf, Command *cmd) {
    char *token = strtok((char *)buf, " ");
    if (token == NULL) {
        return -1;
    }
    
    strcpy(cmd->type, token);
    
    if (strcmp(cmd->type, "CHECK_UPLOAD") == 0 || 
        strcmp(cmd->type, "CHECK_DOWNLOAD") == 0 || 
        strcmp(cmd->type, "DOWNLOAD") == 0 || 
        strcmp(cmd->type, "QUERY") == 0) {
        token = strtok(NULL, " ");
        if (token != NULL) {
            strcpy(cmd->filename, token);
        }
    } else if (strcmp(cmd->type, "UPLOAD") == 0) {
        token = strtok(NULL, " ");
        if (token != NULL) {
            strcpy(cmd->filename, token);
        }
        token = strtok(NULL, " ");
        if (token != NULL) {
            cmd->file_size = atoll(token);
        }
    } else if (strcmp(cmd->type, "BLOCK") == 0) {
        token = strtok(NULL, " ");
        if (token != NULL) {
            strcpy(cmd->filename, token);
        }
        token = strtok(NULL, " ");
        if (token != NULL) {
            cmd->block_id = atoi(token);
        }
        token = strtok(NULL, " ");
        if (token != NULL) {
            cmd->block_size = atoi(token);
        }
        token = strtok(NULL, "");
        if (token != NULL) {
            memcpy(cmd->data, token, cmd->block_size);
        }
    }
    
    return 0;
}
```

#### 7.2.3 获取传输任务

```c
int db_get_task(MYSQL *mysql, const char *filename, TransferTask *task) {
    char query[1024];
    sprintf(query, "SELECT id, file_size, total_blocks, completed_blocks, client_ip, status, md5 FROM transfer_task WHERE filename = '%s'", filename);

    if (mysql_query(mysql, query) != 0) {
        fprintf(stderr, "mysql_query failed: %s\n", mysql_error(mysql));
        return ERROR_DATABASE;
    }

    MYSQL_RES *result = mysql_store_result(mysql);
    if (result == NULL) {
        fprintf(stderr, "mysql_store_result failed: %s\n", mysql_error(mysql));
        return ERROR_DATABASE;
    }

    int num_rows = mysql_num_rows(result);
    if (num_rows == 0) {
        mysql_free_result(result);
        return ERROR_SUCCESS;
    }

    MYSQL_ROW row = mysql_fetch_row(result);
    task->id = atoi(row[0]);
    task->file_size = atoll(row[1]);
    task->total_blocks = atoi(row[2]);
    task->completed_blocks = atoi(row[3]);
    strcpy(task->client_ip, row[4]);
    task->status = atoi(row[5]);
    if (row[6] != NULL) {
        strcpy(task->md5, row[6]);
    } else {
        task->md5[0] = '\0';
    }

    mysql_free_result(result);
    return ERROR_SUCCESS;
}
```

## 8. 多线程并发设计

### 8.1 线程模型

```
主线程 (Main Thread)
    │
    ├─→ accept() 等待客户端连接
    │       │
    │       └─→ 创建客户端处理线程 (Client Handler Thread)
    │               │
    │               ├─→ 接收命令
    │               ├─→ 解析命令
    │               ├─→ 创建传输任务
    │               └─→ 提交任务到线程池
    │
    └─→ 线程池 (ThreadPool)
            │
            ├─→ 工作线程1 (Worker Thread 1)
            │   │
            │   └─→ 处理文件块传输
            │
            ├─→ 工作线程2 (Worker Thread 2)
            │   │
            │   └─→ 处理文件块传输
            │
            └─→ 工作线程N (Worker Thread N)
                │
                └─→ 处理文件块传输
```

### 8.2 线程同步

#### 8.2.1 互斥锁


**使用示例**:


**用途**:

- 通知工作线程有新任务

### 8.3 线程安全

#### 8.3.1 线程安全函数

#### 8.3.2 线程安全策略

1. **共享资源保护**: 使用互斥锁保护共享资源

#### 8.4.1 线程池配置

```bash
# 按Ctrl+C中断

transfer_client upload test_large.txt 127.0.0.1:8888
```

## 14. 故障排除

### 14.1 编译错误

#### 问题1: 找不到mysql.h

**错误信息**:
```
fatal error: mysql/mysql.h: No such file or directory
```

**解决方法**:
```bash
# Ubuntu/Debian
sudo apt-get install libmysqlclient-dev

# CentOS/RHEL
sudo yum install mysql-devel
```

#### 问题2: 找不到openssl/md5.h

**错误信息**:
```
fatal error: openssl/md5.h: No such file or directory
```

**解决方法**:
```bash
# Ubuntu/Debian
sudo apt-get install libssl-dev

# CentOS/RHEL
sudo yum install openssl-devel
```

#### 问题3: 链接错误

**错误信息**:
```
undefined reference to `mysql_real_connect'
undefined reference to `MD5_Init'
```

**解决方法**:
```bash
# 检查Makefile中的LDFLAGS
LDFLAGS = -lpthread -lmysqlclient -lssl -lcrypto
```

### 14.2 运行错误

#### 问题1: 数据库连接失败

## 16. 总结

本项目实现了一个功能完整、性能优异的多线程断点续传网络文件传输工具。通过采用多线程并发、文件分块、断点续传、MD5校验等技术，解决了传统文件传输工具的诸多问题，提供了高效、可靠的文件传输服务。

### 主要特点

1. **多线程并发**: 充分利用多核CPU，提高传输速度
2. **断点续传**: 支持传输中断后从断点处继续传输
3. **大文件支持**: 支持GB级大文件传输
4. **完整性校验**: 使用MD5确保文件传输完整性
5. **传输记录**: 数据库存储传输任务和状态
6. **安全可靠**: 支持各种安全特性和错误处理

### 应用场景

1. **大文件传输**: 视频、音频、压缩包等大文件
2. **跨网络传输**: 不同网络环境下的文件传输
3. **不稳定网络**: 网络波动较大的环境
4. **自动化部署**: 脚本化文件传输
5. **备份同步**: 定期文件备份和同步

本项目不仅满足了基本的文件传输需求，还为未来的扩展和优化提供了良好的基础架构。通过持续改进和功能扩展，可以进一步提高系统的性能、安全性和可靠性，为用户提供更加优质的文件传输服务。
